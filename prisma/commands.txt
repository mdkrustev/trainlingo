1. Инициализиране на Prisma в проекта: Ако нямаш инсталиран Prisma, първо трябва да го добавиш към проекта:
npm install @prisma/client
npx prisma init

2. Форматиране на Prisma схемата: Форматира схемата, за да бъде тя правилно подредена и синтактично вярна:
npx prisma format

2. Създаване на миграция
npx prisma migrate dev --name <migration-name>

3. Приложение на миграцията към базата данни: Миграцията автоматично ще бъде приложена към базата данни. Тази команда съчетава миграцията със самото й изпълнение:
npx prisma migrate dev

4. Преглед на съществуващите миграции: Ако искаш да прегледаш списъка с миграции и техния статус:
npx prisma migrate status

5. Изчистване на всички миграции (за тестови среди): Ако искаш да нулираш базата данни (например за тестови цели), можеш да използваш:
npx prisma migrate reset

6. Генериране на Prisma Client: За да актуализираш и генерираш Prisma Client (който се използва за взаимодействие с базата данни), след миграция или промени в схемата:
npx prisma generate

7.Стартиране на Prisma Studio: За да видиш данните в базата данни през графичен интерфейс, можеш да използваш Prisma Studio:
npx prisma studio

8. Проверка на структурата на базата данни: Ако искаш да видиш SQL командите, които ще бъдат изпълнени от миграцията:
npx prisma migrate resolve --applied <migration-name>
